## 3. API 配置绑定需求（重点）

系统需要暴露 API 配置 `id` 与 `name`。**三层粒度：全局 → 对话 → 插件（按插件 id）**：

- **全局**：用户级默认——各功能键（chat / rag / rerank 等），见 `DOC/03-实现细节.md` §1.2；**插件 API 不在连接设置里配**，见各插件设置页。
- **对话**：会话根 `index.json` 的 `apiPreset`；**同键覆盖全局**；未写的键回退到全局。
- **插件**：在**各插件设置**（全局 / 对话插件 Tab）选 `apiPreset`；**留空**则按 [`DOC/43`](43-plugin-api-binding-audit-checklist.md) §1.1 继承下一层，最终为**全局默认预设**。
- **轮次**：不绑定权威 preset；轮次内仅有**当次调用快照**（审计与复现）。

允许按功能配置的能力包括：

- 对话主模型 / chat
- RAG 生成 / 嵌入侧
- RERANK
- 插件（含 **Historian** `plot-summary`，中文 **剧情纪要**）：**每个需独立 API 的插件**可选用不同 API 预设（会话级叙事摘要由该插件写入资料库，**不再**设全局 `summary` 功能键）

### 3.1 需求规则

- **非插件功能键**：`resolved[key] = 对话.apiPreset[key] ?? 全局(key)`（仍引用全局已登记的 `api_configs`）
- **插件调用（已知 `pluginId`）**：按 `DOC/03-实现细节.md` §1.2.1 **插件解析链**与 §1.3 **包内设定 / 转发**合并，保证不同插件可选用不同 API，且隔离与出站路径符合安全要求
- **不落在「单轮」上**：合并后的有效 preset 用于运行时；历史 `runtime` 快照与当前 preset 解耦
- 配置修改后，不影响历史消息中已保存的快照含义（旧快照仍可解释当时用了什么）
- 删除 API 配置前需校验引用关系（全局与对话引用均需考虑）

### 3.2 插件 API 与安全（V2）

- 插件可携带**包内 API 设定（含密钥）**，**仅覆盖该插件自身**出站调用；**不得**读取或写入全局/对话/其它插件的 API 配置。
- **调用系统已登记 API**：插件/前端**只向服务端提交 `apiConfigId`**（及非密钥业务参数）；**由后端**查表并拼接、提交出站请求。
- **调用自定义 API**：将**本插件作用域内的连接设定**提交服务端；**密钥**仅在服务端按插件槽位存储与合并；**由后端**拼接并提交请求。
- 插件发起的对外请求**必须经后端转发**；前端不持久化明文密钥、不直连外部模型 URL。细则见 `DOC/03-实现细节.md` §1.3–§1.4、§4。
- **对话 API 预设与 Keychain（已实现）**：浏览器 GET 不回明文 key；出站由服务端 resolve。见 **`DOC/25` §15**。
- **settings 存放**：建议**主 `settings.json` + `plugins/<pluginId>/settings.json`**，见 `DOC/03-实现细节.md` §1.4。

### 3.3 备份设定（对话级）

- **触发间隔 N**（每满 N 轮备份一次）、**最大保留份数 K**（3 / 5 / 10 等）均作为**该对话**的配置项存储，与 API preset 一样挂在对话元数据上；**非**应用全局唯一一套（除非另有「新对话默认值」模板）。

## 5. 兼容性需求

- 角色字段兼容 `name/personality/scenario/opening`
- OpenAI 兼容接口优先，支持自定义 baseUrl

### 5.1 提示词组装（lorebook / history / 额外提示词）

- **lorebook**、**history**、**用户的额外提示词**等，在实现上拆为**多块**；字段与 **SillyTavern 提示词管理器** 对齐：**姓名、身份（system/user/assistant）、触发器（生成类型）、位置（聊天中 / 相对）、深度、排序**（细则见 `DOC/03-实现细节.md` §6.6）。
- **位置 = 聊天中**时：**深度 0** 表示在**最后一条消息之后**，**1** 在之前，递增则继续向历史深处回退。
- **位置 = 相对**时：相对大模板锚点或其它提示词；区内深度与 **排序**、身份次序（助手→用户→系统）配合使用。

## 6. 备份与恢复（需求摘要）

- **不设**定时全量备份；**仅**在对话满足轮次条件时触发（见 §3.3）。
- **设定归属**：`N`、`K` 为**每个对话**的配置，随该对话持久化。
- **触发**：在该对话内按累计轮次，每满 **N** 轮触发一次备份任务。
- **内容**：**仅备份本对话 subtree 内自上次备份以来改动过的部分**（增量），非整库全量拷贝。
- **保留**：该对话最多保留 **K** 份增量备份（**K 可配置**，如 3、5、10）；超出时**删除该对话最旧的一份**。
- 完整策略、恢复与安全见 `DOC/03-实现细节.md` §8。

## 7. 首页与会话列表（扩展）

### 7.1 对话列表快查（`chat.index.json` 冗余）

- **动机**：列表 API 仅含角色 **id** 时，前端二次拉角色库才能搜名字/标签，且角色改名后列表滞后；冗余字段写入 **`chats/chat.index.json`**，与 Syncthing 多机一致。
- **冗余字段**（每条 `ChatListEntry`，可选）：**`userName`**（来自会话根 `index.json` 快照）、**`characterNames[]`**（与 `characterIds` 同序）、**`searchTags[]`**（绑定卡 + user persona 的 `tags` 去重合并）。
- **刷新时机**：任意会话 `upsertChatListEntry`（绑卡、改标题、PATCH 等）；角色卡 **PATCH / 删除** 后刷新引用该 id 的列表项；**读列表**时若缺冗余则一次性 enrich 写回。
- **匹配语义**：与角色库 `GET /api/characters?search=` 一致——子串、不区分大小写；范围 = `title` + `userName` + `characterNames` + `searchTags`（不搜全库未绑定角色）。
- **首页双视图**（2026-06）：`/` 顶栏切换对话 / 角色；**设置 → 显示 → 首页** 持久默认视图与角色范围；进入 `/` 或 **从 `/chat/:id` 返回**时均恢复设置默认并清空快查（列表数据仍刷新；「对话消失」由服务端 `reconcileChatListWithDisk` 兜底，与默认视图无关）。

### 7.2 角色视图快查（已实现）

- **`HomeCharacterGrid`**：`GET /api/characters?search=&filter=used|all&sort=name`；`usedInChats` → `filter=used`，`allLibrary` → `filter=all`。
- 点角色卡 → **`CharacterConversationsDialog`** 列出 `chat.index` 中引用该 id 的会话 → 进入 `/chat/:id`。
