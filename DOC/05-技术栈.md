# 技术栈

本文记录项目**已选或默认采用**的技术与工具，便于环境与依赖对齐。若与 `DOC/01-架构设计.md` 冲突，以架构文档为准并同步更新本节。

## 1. 运行时与语言

- **Node.js**：建议 LTS（实现阶段在仓库根 `package.json` 的 `engines` 中锁版本）。
- **包管理**：`npm`（与「首次运行可 `npm install`」的启动脚本一致）。

## 2. 前端

- **框架**：Vue 3（Composition API）
- **状态**：Pinia
- **UI**：Vuetify
- **路由**：Vue Router（`/` 会话列表、`/chat/:conversationId` 对话、`/prompts` 提示词、`/characters` 角色库；设置与 API 连接多为 `App.vue` 内抽屉/模态）
- **国际化**：vue-i18n **v11**（Composition API，`legacy: false`；翻译文件见 `web/src/locales/`，约定见 `DOC/03-实现细节.md` §9）
- **构建**：Vite（Vue3 默认搭配；若仓库初始化时选用其他工具，以实际为准并改本节）
- **HTTP / SSE**：`fetch` 或轻量封装（消费后端 SSE）
- **Markdown / 安全 HTML**：`marked`（GFM）+ `dompurify`（对话气泡）；思维链与正文分流见 `DOC/03-实现细节.md` §11.4

## 3. 后端

- **框架**：Fastify
- **认证**：JWT（无状态，前后端分离）
- **流式**：SSE（Server-Sent Events）向浏览器推送模型 token流
- **对外模型协议**：OpenAI 兼容 HTTP API（`baseUrl` + 密钥 + 模型名）

## 4. 数据与存储

- **主形态**：本地数据目录下的 **JSON 文件**（对话 **chunk**、各级 **`index.json`**）
- **并列数据**：按用户目录 `data/{userId}/` 存放 Prompt（`prompts/index.json` + 各 `preset-*.json`）、角色卡（主存 `{uuid}.png`）、知识书骨架（`lorebooks/`）、API 预设与密钥别名等——以 JSON 或小文件集合存放（路径与 API 见 `DOC/03-实现细节.md` **§12** 及 `data/README.md`）
- **同步**：Syncthing 同步整目录；**单写者、不同时写入**（见架构文档）
- **向量 / 记忆索引**：**LanceDB**（`@lancedb/lancedb`，仅 `server/` 嵌入式）；对话 turn 与（未来）知识库切片分表；详见 `DOC/03-实现细节.md` **§14**
- **数据库**：不依赖 MongoDB 等独立 DB 服务；Lance 为可重建派生数据，**chunk / JSON 为权威**；须经仓储抽象接入

## 5. AI相关能力（MVP）

- **对话**：OpenAI 兼容 Chat Completions（流式）
- **RAG**：嵌入与检索、TopK、**rerank**（各步骤可绑定不同 API 配置，见 `DOC/03-实现细节.md`）
- **Summary**：OpenAI 兼容文本生成（绑定独立 API 配置）

## 6. 工程与交付

- **启动**：根目录 `start.bat`（Windows）与 `start.sh`（Unix），一键拉起所需 Node 进程；首次可检测依赖并执行 `npm install`
- **本地配置**：仓库根 `config.json`（由 `config.example.json` 复制）：`dataDir`、`serverPort`、`webPort`；端口默认 3450/3451（避开 Windows 常见保留段 3326–3425）
- **日志与可观测性**：服务端结构化或分级日志；记录模型耗时、token、RAG 命中明细（需求见 `DOC/02-需求说明.md`）

## 7. 待定（实现时固化）

- Lance 表路径约定（按用户 / 按会话）、`historyTurnCount` / `memoryTopK` 默认值
- 静态知识库 RAG 与对话 memory 是否共用同一 Lance 库实例
- `package.json` 中精确 `engines` 与依赖版本范围

## 8. 已固化（与仓库一致）

- **TypeScript**：`server/` 与 `web/` 均以 TS 为主（`vue-tsc`、`tsc` 校验）。
- **Token 计数**：`js-tiktoken`（仅 **`server/src/token-count.ts`**），Chat Completions 消息格式；`assemble-messages` / `assemble-preview` 与 `contextLength` 裁切共用；**Web 不打包 tiktoken**。
- **提示词组装 / XML**：**仅** `server/src/assemble-prompts.ts`、`server/src/prompt-xml.ts`；Web 通过 `assemble-messages` / `assemble-preview` 获取结果（见 `DOC/03` §15）。history / memory / lore 正文经 **`prepareXmlElementText`**（先解码误存实体再转义），避免 `&quot;` 等双重转义污染模型输入。
- **提示词宏**：`server/src/prompt-macros/`（handler 管线）；**仅服务端**展开 `{{user}}` 等。
- **角色名称排序**：`pinyin-pro`（`server/src/character-name-sort.ts`）；拉丁开头优先于汉字，中文按无声调拼音序。
- **Lance 连接**：`server/src/lance-connection-pool.ts` 进程内 LRU 连接池（上限 32），供 memory / lorebook 向量表复用。
