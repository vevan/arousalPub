## 2. 近期已对齐的实现要点（摘要）

### 2.1 对话数据（服务端）

- 新轮次 **`send`**：`{ userText: string }`；PATCH 使用 **`userText`**。
- 读盘兼容旧 **`sends` + `activeSendIndex`**（`getTurnUserText`）。
- 助手侧：**`receives[]`、`activeReceiveIndex`** 不变。
- **chunk 内轮次不再含 `messages[]`**；调试用 **`chat-prompt.json`**（非 `prompt.json`）+ 请求体 **`debugPrompt`**（`first-turn` / `append-turn` / **`PATCH .../turns/:turnOrdinal`** 再生时）；**`PATCH .../conversations/:id`** 可设 **`promptDebug.maxStored`**；**`GET .../chat-prompt`** 读取。
- **Chunk 切分（已实现）**：`chunk-chain.ts` 按 `turnsPerFile` 滚动 tail；全链读 / memory / messages / PATCH；`syncChunkIndexIfDrifted` 与 **`POST /api/chat/conversations/:id/repair-chunk-index`**；删空 tail 回退 `links.previous`。见 **`DOC/08`**。

### 2.2 前端对话 UI

- **`ChatConversationView`**：`chat_pane`（Grid + `calc(100vh - var(--header-height) - var(--footer-height))`）、`chat-header`（返回、标题、连接提示）。
- **`HomeChat`**：根 **`chat-session`**（flex 列）；**`ChatMessageList`** / **`ChatComposer`** 分别提供 **`chat-body`** / **`chat-footer`**；逻辑在 **`useChatSession`**；勿在 `HomeChat` 外包一层会覆盖 Grid 的 `d-flex`。
- **`App.vue`**：**`<router-view />` 不要挂 `d-flex`**，否则会合并到 `chat_pane` 根上覆盖 `display: grid`。
- 左右抽屉：**`temporary`**、固定宽 280 / 380，不挤压主内容。
- **顶栏/页脚高度**：`--header-height`、`--footer-height` 由脚本测量写入 `document.documentElement`（`style.css` 有兜底）。

### 2.3 富文本

- 正文 / 用户 / 助手主回复：**`renderRichMessageToHtml`**（`marked` + `DOMPurify`，HTML 围栏有条件内嵌）。
- **思维链 reasoning：** **`renderReasoningMarkdownToHtml`**（独立 `Marked` 实例，**不**内嵌 HTML、**不**做 `<codeSample>` 预处理）。

### 2.4 角色库（服务端 + Web）

- **存储**：`server/src/character-storage.ts`、`server/src/character-png.ts`；目录 **`data/{userId}/characters/`**（`getCharactersDir()`，默认用户 `default-user`）；主存 **`{id}.png`**（`id` 为 8 位 hex，见 `DOC/03` §6.7；PNG `chara` chunk）；可选遗留 **`{id}.json`**（首次访问迁移为 PNG）；**`characters/index.json`** 列表索引。
- **提示词**：`server/src/prompts-file.ts`；**`data/{userId}/prompts/index.json`** + **`preset-*.json`**；`GET/PUT /api/prompts`；组装仅服务端 `assemble-prompts.ts`；前端 `web/src/stores/prompts.ts` + API 预览。
- **用户上下文**：`server/src/user-context.ts`（`X-User-Id` / `?user=`，默认 `default-user`）。
- **HTTP**：`GET /api/characters`（**`sort` / `order` / `filterCounts`**）、`POST /api/characters/import`、`POST /api/characters/import-png`、`POST /api/characters`（JSON 或 multipart）、`GET /api/characters/:id/image`、**`GET .../export-png`**、**`GET .../export-json`**、`POST /api/characters/:id/portrait`、`PATCH` / `GET` / `DELETE` **`/api/characters/:id`**（详见 `DOC/03` §12）。
- **前端**：`web/src/views/CharactersView.vue`，路由 **`/characters`**；顶栏「角色」在 `App.vue`；排序 UI 在搜索框前，侧栏三项 **`filterCounts`**。
- **「已使用」**：由 `readChatList` + 各会话 **`characterId` / `characterIds`** 聚合（**`resolvedCharacterIds`** 含 legacy 单卡回退），非角色文件内字段。
- **名称排序**：`server/src/character-name-sort.ts` + **`pinyin-pro`**；默认 **`name` + `asc`**（拉丁名如 `vevan` 排在中文拼音名前）。

### 2.5 会话列表与 XML（2026-05-26）

- **`ConversationListView`**（`/`）：卡片展示 user + 主角色头像；新建弹窗 **`createTitleDraft`**；列表头 **`conversationList.count`**。
- **思维链复制**：`ChatTurnAssistant.vue` reasoning summary 复制按钮。
- **记忆重建弹窗**：仅 **`memoryEmbeddingModel` 已存在**且与全局 embedding 不一致时触发（新建会话跳过）。
- **Memory XML / History 消息链**：Memory 仍用 `<memory>` XML（`prepareXmlElementText` 防双重转义）；**近期 history 组装**为 `user`/`assistant` 多条（`turnsToHistoryMessages`，见 `DOC/03` §14）。
- **Lance 连接池**：`lance-connection-pool.ts`（LRU，上限 32 URI）。
